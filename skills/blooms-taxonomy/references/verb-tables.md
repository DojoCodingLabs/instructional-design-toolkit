# Builder's Bloom's — Action Verb Quick Reference

Use these verbs when writing learning objectives. The verb determines the cognitive level. If you're writing an objective and aren't sure what level it targets, find the verb in this table.

## Level 1: Recognize (Remember)

The student can identify it when they see it.

| Category | Verbs |
|----------|-------|
| **Identification** | identify, recognize, name, label, locate, spot |
| **Recall** | list, recall, state, define, describe (basic), match |
| **Selection** | select, choose (from given options), point to, find |

**Assessment fit**: Multiple-choice, matching, "which one is X?", terminology checks.

**Example objectives**:
- "You'll identify the three main component types in React."
- "You'll recognize when a use case calls for server-side rendering."
- "You'll name the key files in a standard Next.js project structure."

---

## Level 2: Explain (Understand)

The student can describe how and why it works.

| Category | Verbs |
|----------|-------|
| **Description** | explain, describe, summarize, paraphrase, restate |
| **Comparison** | compare, contrast, distinguish, differentiate |
| **Interpretation** | interpret, predict, infer, translate, illustrate |
| **Classification** | classify, categorize, group, organize |

**Assessment fit**: Short-answer questions, "explain in your own words," comparison tables, "what would happen if...?"

**Example objectives**:
- "You'll explain why environment variables should never be committed to Git."
- "You'll compare REST and GraphQL trade-offs for your use case."
- "You'll predict what happens when you deploy without a build step."

---

## Level 3: Build (Apply)

The student can use a tool/pattern to produce a working result.

| Category | Verbs |
|----------|-------|
| **Construction** | build, create (guided), construct, implement, produce |
| **Execution** | execute, run, deploy, launch, set up, configure |
| **Usage** | use, apply, operate, practice, demonstrate |
| **Problem-solving** | solve, calculate, complete, modify (guided) |

**Assessment fit**: Guided challenges, BUILD sections, "make this work," follow-the-pattern exercises.

**Example objectives**:
- "You'll build a landing page using the component library from this module."
- "You'll deploy your project to Vercel with environment variables configured."
- "You'll set up a CI/CD pipeline that runs tests on every push."

---

## Level 4: Debug & Evaluate (Analyze)

The student can break things down, trace causes, and assess quality.

| Category | Verbs |
|----------|-------|
| **Debugging** | debug, trace, diagnose, troubleshoot, isolate |
| **Evaluation** | evaluate, assess, test, audit, review, inspect |
| **Deconstruction** | deconstruct, break down, dissect, examine, map |
| **Comparison** | compare (outputs), differentiate (quality), benchmark |
| **Root cause** | identify (root cause), trace (origin), attribute, explain (failure) |

**Assessment fit**: "Find the bug" challenges, code review exercises, AI output evaluation, quality rubric application.

**Example objectives**:
- "You'll debug a failing deployment by tracing the error from logs to source."
- "You'll evaluate AI-generated code for correctness, security, and maintainability."
- "You'll audit your project's dependencies for known vulnerabilities."

---

## Level 5: Decide (Evaluate)

The student can choose between approaches and justify the trade-off.

| Category | Verbs |
|----------|-------|
| **Decision** | choose, decide, select (with justification), pick |
| **Justification** | justify, defend, argue for, rationalize, support |
| **Judgment** | judge, rank, prioritize, rate, weigh |
| **Critique** | critique, challenge, question, counter-argue |
| **Recommendation** | recommend, propose, advise, advocate |

**Assessment fit**: Architecture decision records, "which approach and why?", trade-off analysis, capstone planning, peer review.

**Example objectives**:
- "You'll choose between a monorepo and multi-repo setup and defend your decision."
- "You'll prioritize your backlog based on user impact and technical complexity."
- "You'll recommend a database architecture for your capstone with documented trade-offs."

---

## Level 6: Ship (Create)

The student designs and deploys something original.

| Category | Verbs |
|----------|-------|
| **Design** | design, architect, plan, blueprint, spec out |
| **Creation** | create (original), invent, compose, author, develop |
| **Shipping** | ship, deploy (original), launch, publish, release |
| **Integration** | integrate, combine, synthesize, assemble, orchestrate |
| **Innovation** | innovate, improve, extend, reimagine, remix |

**Assessment fit**: Capstone projects, portfolio pieces, original tool creation, public deployments.

**Example objectives**:
- "You'll ship a SaaS product that solves a problem you identified through user research."
- "You'll design and deploy an AI agent that automates a real workflow."
- "You'll architect a full-stack application and defend every technology choice."
